# Fundamentos de Ingeniería de Software 🏗️

## 1. Problemática del Desarrollo de Software
Desarrollar software es una tarea **compleja por naturaleza**. No se trata solo de codificar, sino de gestionar una serie de desafíos críticos:

* **Complejidad y Plazos:** Es extremadamente difícil planificar con exactitud y cumplir con los tiempos estipulados.

* **Planificación y Plazos:** Es difícil planificar con exactitud y cumplir con los plazos (deadlines) y presupuestos iniciales.

* **Defectos del Producto:** El software suele presentar fallos que exigen un seguimiento constante.

* **Comunicación con el Cliente:** Tiene que haber una comunicación activa con el cliente para alinear expectativas; de lo contrario, el proyecto fracasa. 

* **Evolución del SW:** El software nunca está "terminado"; debe evolucionar, cambiar y adaptarse constantemente para no quedar obsoleto.

## 2. Particularidades del Software
A diferencia de productos tangibles (como un edificio), el software posee características únicas:

* **Intangibilidad:** Al ser un producto no físico(que no se puede tocar), es difícil visualizar su estado de avance.

* **No es Monofactura:** No es un proceso repetitivo de fábrica; implica un desarrollo creativo e intelectual único en cada caso (mezcla procesos creativos y técnicos).

* **Involucra Personas:** * **Basado en el intelecto y las personas:** El éxito depende directamente del talento, la creatividad y la comunicación del equipo.

* **Complejidad del Producto:** * **Cambio Tecnológico:** Actualizaciones constantes e incompatibilidades frecuentes.

    * **Satisfacción del Cliente:** Es difícil cubrir todas las necesidades. Se recomienda el uso de **documentos tipo contrato firmado** por ambas partes para formalizar cambios.

    * **Sistemas Heterogéneos:** Necesidad de integrarse con diferentes plataformas y sistemas que no son iguales entre sí.

    * **Sistemas Críticos:** Su fallo puede poner en riesgo vidas o economías (ej. software médico, control de tráfico aéreo o sistemas financieros).



### El Efecto Iceberg 🧊
En el desarrollo, lo que el usuario ve es mínimo. El **Mantenimiento** es la base del iceberg: es la parte más grande, la que más dura y la más cara. Por eso, es fundamental saber cobrarlo y gestionarlo.



## 3. Contexto del Negocio
El software debe generar valor para la organización. Factores clave del negocio incluyen:
* **Estratégico:** Es un desafío implementar software en empresas que todavía funcionan "todo en papel".

* **Time-to-market:** Salir rápido al mercado es una ventaja competitiva.

* **Costos de Desarrollo y Mantenimiento:** El mantenimiento es mucho más caro que el desarrollo inicial. 

El software es "muy visible" y afecta procesos críticos del negocio.


## 4. De "Arte" a "Ingeniería"
El objetivo es **profesionalizar** la actividad:
1.  **Transformar el Arte en Ingeniería:** Pasar de la intuición a métodos científicos, matemáticos y técnicos.

2.  **Sistematizar:** Crear actividades ordenadas pero que dejen espacio para la **creatividad**.

3.  **Disciplina:** Para tener éxito se requiere un enfoque riguroso de ingeniería.

> **Objetivo:** Profesionalizar el desarrollo, transformándolo de un "arte" manual a una "ingeniería" basada en principios científicos, matemáticos y técnicos.

## 5. Definición y Pilares (El Trípode)
La Ingeniería de Software es la aplicación de un enfoque **sistemático, disciplinado y cuantificable** al desarrollo, operación y mantenimiento del software.

### Capas de la Ingeniería:
* **Procesos:** El marco de trabajo que sostiene todo el proyecto.

* **Métodos:** El "cómo" técnico; aplicación de principios de ciencias y matemáticas para la producción económica de software.

* **Herramientas:** Soporte automático para mejorar la eficiencia (ej. VS Code, Git).



## 6. Curva de Estabilización
En los **primeros 3 meses** de vida de un software es normal recibir muchas llamadas para arreglar fallos. El objetivo de la ingeniería es que, mediante una producción económica y disciplinada, esa curva de errores baje y el sistema se vuelva estable y fácil de modificar.
